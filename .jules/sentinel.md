## 2024-06-15 - Timing Side-Channels & Uncaught Exceptions in Express Signature Verification
**Vulnerability:** Express header parsing and `crypto.timingSafeEqual` were interacting dangerously. Missing explicit string validation on `req.headers` allowed potential TypeErrors to crash the server when arrays were passed to `Buffer.from()`. Furthermore, using `try...catch` around `crypto.timingSafeEqual` introduces a minor timing side-channel and obscures input-validation issues.
**Learning:** `req.headers` can be strings, arrays of strings, or undefined. `Buffer.from(..., 'hex')` strictly expects strings/buffers and throws on arrays. Additionally, `crypto.timingSafeEqual` throws if buffer lengths mismatch. Relying on `try...catch` handles the errors but introduces performance hits and timing variances.
**Prevention:** Always explicitly validate that Express headers are strings (`typeof header === 'string'`) before passing them to native crypto functions. Always explicitly compare buffer lengths (`if (a.length !== b.length) return false;`) before calling `timingSafeEqual` instead of catching the resulting error.
