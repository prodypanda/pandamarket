## 2025-02-28 - SQL Injection via Interval String Interpolation\n**Vulnerability:** Found a raw SQL query using string interpolation for an interval value (e.g., `INTERVAL '${val} days'`) and an unparameterized array payload.\n**Learning:** This existed because of dynamic date offset logic combined with inline `IN (...)` clauses.\n**Prevention:** Use PostgreSQL array parameterization `ANY(::text[])` and safe integer casting with multiplication for dynamic intervals `(::int * INTERVAL '1 day')`.
