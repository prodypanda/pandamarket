## 2026-07-04 - DoS Vulnerability via Crypto timingSafeEqual Exception

**Vulnerability:** Flouci and Konnect webhook signature verification endpoints were prone to Denial of Service (DoS). The code relied on `try-catch` blocks and called `crypto.timingSafeEqual(Buffer.from(signature, 'hex'), Buffer.from(expected, 'hex'))` directly. Furthermore, it cast `req.headers['x-...-signature']` as `string | undefined` without actually checking if it was a string or string array.
**Learning:** `crypto.timingSafeEqual` throws a `RangeError` if the passed buffers have different lengths. In Express, header variables can be arrays, and passing an array to `Buffer.from` causes an exception. Using `try-catch` to handle expected invalid inputs (such as spoofed/invalid signature payloads) causes severe CPU overhead and stack trace generation on every request, making it easily exploitable for DoS under spam.
**Prevention:** Always explicitly validate that request header values used in crypto functions are single strings (`typeof signature === 'string'`). Convert the inputs to buffers and explicitly verify that `sigBuf.length === expBuf.length` before calling `crypto.timingSafeEqual` to avoid any exceptions.
