## 2026-07-05 - DoS Vulnerability in `crypto.timingSafeEqual` Buffer Comparison
**Vulnerability:** Application crashed or exhausted CPU resources when webhook verification headers (`x-flouci-signature` or `x-konnect-signature`) provided signatures of different lengths than expected, throwing unhandled `RangeError` from `crypto.timingSafeEqual`, which was inside a frequent `try-catch` block leading to a Denial of Service (DoS) vulnerability.
**Learning:** Node.js `crypto.timingSafeEqual` strictly requires input buffers to have identical byte lengths, and relying on exception-based control flow (`try-catch`) for handling expected invalid inputs like malformed cryptographic signatures is a performance anti-pattern. Generating stack traces for frequent exceptions during spam attacks can exhaust CPU resources.
**Prevention:** Always explicitly validate that `buf1.length === buf2.length` prior to calling `crypto.timingSafeEqual`. Explicitly validate header values are single strings before casting/using `Buffer.from()` to prevent unhandled TypeErrors.
