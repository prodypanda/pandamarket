## 2024-05-31 - SQL Injection via Unsafe PostgreSQL INTERVAL String Interpolation
**Vulnerability:** A raw string interpolation was used to construct a PostgreSQL `INTERVAL` clause (`INTERVAL '${older_than_days} days'`) within a `DELETE` query. In addition, an `IN` clause was dynamically constructed (`IN (${roleFilter})`) where `roleFilter` was derived from user input without using parameterized queries, though the string interpolation on `roleFilter` was loosely constrained in this specific endpoint.
**Learning:** Even when inputs are validated via Zod (like `older_than_days`), string interpolating them into SQL fragments bypassing parameterized query bindings violates defense-in-depth principles. Attackers bypassing validation (or if validation is weakened later) could perform critical SQL injections.
**Prevention:** For dynamic intervals in PostgreSQL, use mathematical scaling on parameterized values (e.g. `($1::int * INTERVAL '1 day')`). For dynamic `IN` clauses, use array parameter bindings with `ANY` (e.g. `ANY($2::text[])`).
