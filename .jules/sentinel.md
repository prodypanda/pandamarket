## 2026-06-19 - Unhandled Exception in timingSafeEqual causing DoS Risk
**Vulnerability:** crypto.timingSafeEqual throws a RangeError when the input buffers have different byte lengths. Previously, this exception was sometimes caught, but upstream input parsing (Buffer.from(signature, 'hex')) could also produce buffers of unexpected sizes when the signature length or encoding is mismatched, leading to uncaught errors or unintended flow.
**Learning:** In Node.js, `Buffer.from(string, 'hex')` will silently parse until the first invalid char or produce a buffer of a different size than expected. When passing these directly to `crypto.timingSafeEqual`, a length mismatch throws a `RangeError`. This unhandled exception acts as a Denial of Service (DoS) vulnerability.
**Prevention:** Always verify that both `Buffer` objects are of the exact same length (e.g. `buf1.length !== buf2.length`) before calling `crypto.timingSafeEqual` and explicitly validate that request header values are single strings (`typeof header === 'string'`) instead of arrays.
